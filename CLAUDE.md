# Gesture Show! 開発ガイド

## 起動

プロジェクトルートで次を実行し、`http://localhost:8000` を開きます。

```bash
python -m http.server 8000
```

`file://` 直開きではカメラ API や ES Modules の扱いがブラウザによって異なるため、開発・検証では再現性を保てる localhost の静的サーバーを使用します。カメラを使えない環境でも、マウス／タッチ操作で画面と同期を確認できます。

## 2 タブでの同期確認

1. 同じブラウザで `http://localhost:8000` を 2 タブ開きます。
2. 先に開いたタブが送信機、後から開いたタブが受信機になることを同期バッジで確認します。
3. 送信機でカテゴリとお題を選択し、受信機に選択中のお題が反映されることを確認します。別のお題を選ぶと前の選択が外れることも確認します。
4. 送信機でリセット、合掌または戻るボタンによる HOME 遷移を行い、受信機へ同期されることを確認します。
5. 受信機の「操作権をとる」で役割を交代し、新しい送信機で同じ操作ができることを確認します。

## アーキテクチャ地図

- `index.html`: HOME/GAME、各操作部、MediaPipe と PeerJS の読み込みを定義します。
- `app.js`: 起動、DOM イベント配線、UI と同期の接続を担当します。
- `style.css`: ネオン調の画面レイアウト、レスポンシブ表示、状態別スタイルを定義します。
- `src/state.js`: 共有状態と DOM 参照を保持します。
- `src/data.js`: 10 カテゴリ、各 33〜46 語のお題データを定義します。
- `src/ui.js`: HOME/GAME 描画、単一選択、リセット、画面遷移を担当します。
- `src/cursor.js`: カーソル補間、ホバー、即時グーによる疑似クリックを担当します。
- `src/gestures.js`: MediaPipe Hands、手の認識、グー・合掌、骨格描画を担当します。
- `src/sync.js`: PeerJS の送信機／受信機判定、状態配信、操作権交代を担当します。

## 開発時の注意

- 固定 ID `gesture-game-master-sender` は送信機の判定に使います。ID を変更する場合は、送信側・受信側で同じ定数を参照するように保ってください。
- PeerJS は接続切断、再接続、役割交代のイベント順が非同期です。接続やタイマーを増やさず、古い Peer／接続を解放してから再初期化してください。
- 設計規約として受信機は表示専用として扱い、状態変更の入口には `syncRole` の role ガードを置いてください。受信機の描画は、送信機から届いた同期状態によってのみ更新します。
- `python -m http.server 8000` を止める際は、そのプロセスだけを終了してください。通常どおり差分を確認し、必要なテストを実行してからコミットしてください。
