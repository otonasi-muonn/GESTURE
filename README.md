# Gesture Show!

カメラに映した手、またはマウス／タッチで操作できるジェスチャーゲーム用の回答表示ボードです。出題を操作する送信機と、答えを表示する受信機を PeerJS で P2P 同期します。

## 操作

操作語彙は次の 3 つです。

| 操作 | 入力 | 動作 |
| --- | --- | --- |
| カーソル移動 | 人差し指の先端 | 指先に追従して画面上の対象へカーソルを移動します。 |
| 決定 | 人差し指だけを伸ばす | カーソル下のカテゴリ・お題・ボタンを、検出フレームごとに選択します。待ち時間はありません。 |
| 戻る | 親指と人差し指だけを伸ばす | ゲーム画面からホームへ、1ポーズにつき1回だけ戻ります。ポーズを崩すと再び戻れます。 |

お題はラジオボタンと同じ**単一選択**です。別のお題を選ぶと、前の選択は自動的に解除されます。カメラが利用できない場合も、マウス／タッチで全操作を続けられます。

## お題カテゴリ

10 カテゴリに、それぞれ 33〜46 語のお題を収録しています。

| カテゴリ | 語数 |
| --- | ---: |
| 動物 | 46 |
| 職業 | 46 |
| 食べ物（食べる・作る） | 46 |
| 楽器 | 33 |
| 日用品・道具 | 46 |
| スポーツ用品 | 43 |
| 美容・健康グッズ | 42 |
| IT・ゲーム関連 | 40 |
| おもしろ系・キャラクター | 46 |
| 乗り物 | 33 |

## 送受信の同期と操作権

最初に起動した端末が送信機となり、カメラとジェスチャーでカテゴリ・お題・リセット・戻るを操作します。後から開いた端末は受信機として同じ状態を表示します。受信機の「操作権をとる」から、送信機と受信機の操作権を交代できます。

同期には固定 ID を使った PeerJS の P2P 接続を使用します。接続可否はネットワーク、NAT、ファイアウォール、ブラウザ環境に依存します。固定 ID は認証情報ではなく、ID を知る接続者は操作権交代を要求できます。信頼できる参加者だけの利用を前提とし、公開ネットワークでの無人運用や機微情報の表示には使用しないでください。

## 起動方法

カメラや ES Modules の扱いに関するブラウザ差を避け、再現性のある環境で確認するため、localhost の静的サーバーから開くことを推奨します。

```bash
python -m http.server 8000
```

起動後、ブラウザで [http://localhost:8000](http://localhost:8000) を開きます。初回はカメラ利用を許可してください。カメラがなくても、マウス／タッチで確認できます。

## 構成

- HTML / CSS / JavaScript の静的 ES Modules 構成です。
- 手の検出には MediaPipe Hands、送受信同期には PeerJS 1.4.7 を使用します。
- 外部音声ファイルを使わず、Web Audio API で効果音を生成します。
