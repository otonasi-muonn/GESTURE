# Gesture Show! 指先入力仕様

## 目的

人差し指の先端でカーソルを直接動かし、片手の指構成だけで決定と戻るを操作できるようにする。

設計上の正典は `docs/decisions/20260719_gesture_pointer_actions.md` とし、本書はその受け入れ仕様を定義する。

## 利用者から見える動作

1. カメラが手を検出すると、カーソルは人差し指の先端へ追従する。
2. 人差し指だけを伸ばしている間は、そのままカーソルを移動できる。
3. その状態でカーソルが操作可能なボタンへ重なると、検出フレームごとに決定入力が送られる。重なった瞬間から反応し、待ち時間はない。
4. sender の GAME で人差し指と親指だけを伸ばす成立エッジでは、戻る操作が即時に1回発火する。
5. HOME・viewer・loading でも戻るポーズのラッチは更新されるが、画面遷移は発火しない。戻るポーズを維持したまま sender/GAME へ変わっても、後から戻る操作は発火しない。検出中のすべての手で戻るポーズが不成立になった後に、再び戻るポーズを作ると次の1回が発火する。片方の手だけがポーズを崩しても、別の手が戻るポーズを維持している間は再武装しない。

## 判定の排他性

| ポーズ | 親指 | 人差し指 | 中指 | 薬指 | 小指 |
|---|---:|---:|---:|---:|---:|
| 決定 | 曲げる | 伸ばす | 曲げる | 曲げる | 曲げる |
| 戻る | 伸ばす | 伸ばす | 曲げる | 曲げる | 曲げる |

2つのポーズは親指の状態で排他的に判定する。どちらにも一致しない指構成では、ジェスチャーによる決定と戻るを発火しない。マウスとタッチによる操作は維持する。

## 入力状態

- 決定にはラッチ、保持時間、フレームヒステリシスを設けない。
- 戻るには `ready` / `latched` の2状態を持たせる。
- role と screen に関係なく、`ready` でいずれかの手に戻るポーズが成立したら `latched` へ移る。
- 検出中のすべての手で戻るポーズが不成立になった時だけ `ready` へ戻す。片方の手だけを崩しても、別の手が戻るポーズなら `latched` を維持する。
- 画面遷移と音は `state.syncRole === 'sender' && state.currentScreen === 'GAME' && shouldTrigger` の時だけ実行する。非 GAME または非 sender で成立した戻るポーズはラッチされるが、後から role/screen が変わっても維持中のポーズでは発火しない。

## 受け入れ条件

- カーソル位置の入力ランドマークが人差し指先端である。
- 人差し指だけのポーズを維持すると、対象への決定入力が複数フレーム連続で発生する。
- 人差し指と親指だけのポーズを維持しても、戻るは1回しか発生しない。
- HOME・viewer・loading で成立した戻るポーズはラッチされ、維持したまま sender/GAME へ切り替えても戻るは発生しない。
- 検出中のすべての手で戻るポーズを崩してから再度作ると、戻るがもう1回発生する。
- グーだけでは決定されず、両手を合わせるだけでは戻らない。
- UI、README、監査文書に旧操作説明が残らない。
- 既存の同期、カメラ停止、マウス／タッチ操作に回帰がない。

## 対象外

- 決定入力の間隔制限やクールダウン。
- ボタン上での滞留時間による決定。
- 新しい外部ライブラリまたは学習済み Gesture Recognizer の導入。
- 戻る以外のジェスチャーへの単発ラッチ追加。
