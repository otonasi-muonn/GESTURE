# Gesture Show! ブラッシュアップ監査・実施基準

## 目的

Gesture Show! を、ポインティング・決定・戻るの最小操作だけで迷わず使え、デスクトップではスクロールせず、送受信の切断や操作権交代にも耐える回答表示ボードへ整える。

価値軸は「操作の少なさ」「1 画面内での状態把握」「同期の自己回復」である。固定同期 ID、MediaPipe Hands、PeerJS 1.4.7 と、待ち時間のない指先入力を維持する。

## 操作設計

以下は、`docs/decisions/20260719_gesture_pointer_actions.md` を唯一の設計正典とする操作設計要約である。

| 操作 | 入力 | 発火方式 |
|---|---|---|
| ポインティング | 人差し指の先端 | 手の検出中にカーソルを追従させる |
| 決定 | 親指を含む他4指を曲げ、人差し指だけを伸ばす | 送信機でカーソル下の対象を検出フレームごとに押す |
| 戻る | 親指と人差し指だけを伸ばす | 全 `onResults` でrole/screenに関係なくラッチを更新し、送信機の GAME での成立エッジだけが即時1回の画面遷移と音を発火する |

決定には待ち、ラッチ、保持時間を設けない。戻るだけを単発にし、維持中の再発火を防ぐ。戻るはHOME・viewer・loadingでもラッチされ、ポーズを保持したまま sender/GAME へ変わっても後から画面遷移しない。検出中の全手で戻るポーズが不成立になった時だけ再武装する。マウス／タッチ操作はカメラの有無にかかわらず維持する。

選択方式の研究では Dwell の精度が高く Pinch の誤りが多いという報告があるが、既存の操作習慣とユーザー決定を優先する。学習済み Gesture Recognizer への移行は将来候補に留める。

## 現状アーキテクチャ

| ファイル | 責務 |
|---|---|
| `index.html` | HOME/GAME、カーソル、カメラ背景、MediaPipe/PeerJS 読み込み |
| `app.js` | 起動、イベント配線、同期コールバック |
| `src/state.js` | グローバル状態と DOM 参照 |
| `src/data.js` | 10 カテゴリの語彙 |
| `src/ui.js` | 描画、遷移、単一選択、リセット |
| `src/cursor.js` | 人差し指先端カーソルの補間、hover、検出フレームごとの決定入力 |
| `src/gestures.js` | MediaPipe、指構成判定、戻る単発ラッチ、骨格描画 |
| `src/sync.js` | 固定 ID の送信機／受信機判定、状態配信、操作権交代 |
| `style.css` | ネオン系 UI とレスポンシブレイアウト |

先に固定 ID `gesture-game-master-sender` を取得した端末が送信機となり、カメラと操作を担当する。後続端末は受信機となり、送信機の状態を表示する。受信機は画面ボタンから操作権交代を要求できる。

## 監査結果と実施方針

### 同期

- `close`、`error`、初期化エラーからの再試行を単一の再起動スケジューラへ統合し、再生成前に旧 Peer、接続、タイマーを破棄する。
- シグナリング切断時は既存 Peer の `reconnect()` を試す。
- 初回状態は固定待ちではなく `conn.open` 後に送信する。
- 操作権交代は固定 600ms 待ちを廃止し、送信機 ID の取得を上限付きで再試行する。失敗時は受信機へ安全に降格する。
- 役割変更後は UI を再描画し、送信機から受信機へ移る際は gesture モジュールの停止 API で Camera、Hands、MediaStreamTrack を止める。
- 受信機からのカテゴリ、単語、リセット、戻るのローカル変更を遮断し、同期受信による画面遷移は許可する。
- 固定 master ID は 1 定数へ集約する。

### レイアウトと CSS

- デスクトップは `100dvh` の flex レイアウトと `min-height: 0` を用い、HOME/GAME とも主領域を残り高さへ収める。
- 固定 `vh` の積み上げをやめ、`clamp()` と可変グリッドで高さ 650px まで 1 画面完結を目指す。
- 768px 未満は既存どおり必要時にスクロールを許容する。
- 長い日本語はカード内で折り返す。10 カテゴリすべてに色を割り当て、hex 値を `rgba()` に渡す無効宣言を `color-mix()` へ置換する。
- HTML から参照されない旧ガイド・統計・ステータス用 CSS を削除し、重複する同期バッジ定義を統合する。

### ジェスチャー

- カーソルは人差し指先端を基準にし、親指を含む各指の伸展／屈曲を比較して決定・戻るポーズを排他的に判定する。
- 決定は送信機で人差し指だけのポーズ中、検出フレームごとにカーソル下の操作可能な対象へ送る。戻るの成立エッジとラッチは全 `onResults` でrole/screenに関係なく計算し、送信機 GAME の `shouldTrigger` だけが画面遷移と音を1回実行する。
- カメラ失敗時は「カメラ利用不可（マウス／タッチ操作可）」を画面へ表示する。
- Camera、Hands、映像トラックを保持し、冪等な `stopMediaPipe()` で停止する。
- 未使用関数、引数、import、到達不能セレクタ、重複 import を削除する。

### 文書と配布資産

- README を現在の 10 カテゴリ、単一選択、指先入力、送受信同期、操作権交代へ更新する。
- 空の `README` を削除し、開発起動・2 タブ同期確認・アーキテクチャ地図を `CLAUDE.md` に記録する。
- MediaPipe legacy CDN は正確なフルバージョンへ固定し、PeerJS 1.4.7 は維持する。
- favicon は外部ファイルを増やさず、インライン SVG とする。

## 技術判断

- 既存の静的 ES Modules 構成を維持する。ビルド工程や外部依存を増やす利益が今回の安定化目的に見合わないため。
- UI は CSS の flex/grid を修正する。DOM 全面刷新は同期・カーソルのセレクタ契約を壊すため採らない。
- 同期は PeerJS の既存イベントモデル内でライフサイクルを明示化する。別ライブラリやルーム管理は対象外とする。
- 自動検証には既存環境の構文検査と、外部依存を追加しない純粋関数テスト、ブラウザ 2 タブ E2E を組み合わせる。

固定 ID は参加者の認証や操作権交代要求の認可を行わない。ID を知る接続者が交代要求を送れるため、本版は信頼できる参加者だけの利用を前提とし、公開ネットワークでの無人運用は対象外とする。認証、ルームコード、送信機側の承認フローは今回の明示的な変更対象外である。また CDN の完全版固定は再現性確保を目的とし、SRI やローカル配布による供給元の完全性保証は対象外とする。

変更は複数モジュールへ及ぶが、共有データや個人情報を扱わずローカルで容易に戻せるため、レビュー深度は medium とする。

## ゲート別受け入れ条件

1. 文書: README、開発手順、コメントが実装と一致する。
2. UI: 1920x1080、1366x768、高さ 650px のデスクトップで HOME/GAME の縦スクロールが発生しない。
3. ジェスチャー: 人差し指先端カーソル、指構成の排他判定、検出フレームごとの決定、戻るの単発・再武装を自動検証し、カメラなしでもマウス／タッチ操作を継続できる。
4. 同期: 2 タブで送受信判定、カテゴリ、単語、リセット、戻る、操作権交代が同期し、再接続タイマーが増殖しない。
5. 品質: JavaScript 構文検査、該当自動テスト、console、視覚確認、多角レビュー、批判レビューを通す。

実カメラでの手の向き差と操作感だけは環境依存のため、最終的にユーザー確認項目として残す。

## 参考知見

- Tier 1 同等: ユーザー決定「人差し指先端カーソル、決定は人差し指だけ、戻るは親指と人差し指だけ」を採用する。
- Tier 1: Google MediaPipe Hands。既存の手ランドマークを用い、外部の学習済み Gesture Recognizer へは移行しない。
- Tier 2: ACM ISS 2022 の選択方式比較。Dwell の高精度と Pinch の誤り傾向を参考にしたが、今回の入力方式変更は行わない。

## 実施結果（2026-07-19）

- 文書、UI、ジェスチャー、同期をゲート単位で実装し、外部テスト依存を追加せず Node 標準テスト 28 件を整備した。
- `app.js`、`src/*.js`、`tests/*.mjs` の構文検査、全 28 テスト、`git diff --check` が成功した。
- HOME と送信機 GAME は 1920x1080、1366x768、1366x650、受信機 GAME は 1920x1080、1366x650 で、文書高さと表示高さが一致し縦スクロールがないことを確認した。46 語の最大カテゴリでもカード内の文字切れはなかった。
- 実 PeerJS の一時テスト ID を用いた 2 タブ検証で、送受信役割の確立、カテゴリ、単語、リセット、戻るの同期と、操作権交代後の役割反転を確認した。一時 ID の差し替えは配信ハーネス内だけで行い、製品コードの固定 ID は変更していない。
- 通常配信ページは 10 カテゴリを描画し、console の warning/error が 0 件だった。操作権交代時の初回接続ギャップは、再試行を維持しつつ回復可能な `peer-unavailable` だけを error 扱いしないようにし、接続成立後と別種の障害が引き続き error になる境界をテストした。
- カメラ初期化失敗、部分初期化失敗、永久に保留する開始、旧開始の遅延完了、停止中の再初期化、二重停止を自動検証した。Camera ごとに専用 video／stream を所有させ、旧世代が新 preview へ干渉しないようにし、失敗時もマウス／タッチ操作へ退避できる表示を維持した。

> この節のグー3フレーム／解除2フレームと合掌戻るは当時の検証履歴であり、2026-07-19の指先ジェスチャー入力変更によって置き換えられた。
