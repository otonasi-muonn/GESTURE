# ジェスチャー入力割り当ての変更

- 状態: 承認済み
- 決定日: 2026-07-19
- レビュー深度: medium

> このADRを指先ジェスチャー入力の唯一の設計正典とする。受け入れ条件は入力仕様、実装の手順と履歴は実装計画で管理する。

## 背景

従来は手の中心付近でカーソルを動かし、グーで決定、両手の合掌で戻っていた。ユーザーは、指先とカーソルの対応を直感的にし、片手だけで決定と戻るを操作できる割り当てを選択した。

## 決定

| 操作 | 入力 | 発火方式 |
|---|---|---|
| カーソル移動 | 人差し指の先端 | 手を検出している間、指先を追従する |
| 決定 | 人差し指だけを伸ばす | ポーズ中、カーソル下の対象へ検出フレームごとに決定入力を送る |
| 戻る | 人差し指と親指だけを伸ばす | 全 `onResults` でrole/screenに関係なく成立エッジとラッチを更新し、`sender` の `GAME` での成立エッジだけが画面遷移と音を1回発火する |

「指だけを伸ばす」は、記載した指が伸び、その他の指が曲がっている状態を指す。決定は人差し指の状態を維持したままカーソルを動かす使い方を正典とし、ボタン上を通過しただけでも決定される。保持時間や決定用の再発火ロックは設けない。

戻るの成立エッジと `ready` / `latched` の更新は、role と screen に関係なく全 `onResults` で行う。画面遷移と音は `state.syncRole === 'sender' && state.currentScreen === 'GAME' && shouldTrigger` の時だけ実行する。したがって、HOME・viewer・loading で戻るポーズを成立させても画面は遷移しないがラッチされ、ポーズを保持したまま sender/GAME へ変わっても後から発火しない。再武装は、検出中のすべての手で戻るポーズが不成立になった時だけ行う。片方の手だけがポーズを崩しても、別の手が戻るポーズならラッチを維持する。

従来のグーによる決定と両手の合掌による戻るは廃止する。画面内の操作説明も新しい割り当てへ統一する。

## 理由

- 人差し指の先端と画面カーソルが直接対応し、狙う位置を把握しやすい。
- 決定を人差し指ポーズ中の連続入力にすることで、ポーズを切り替えずに移動と選択を行える。
- 戻るだけを単発にすることで、複数画面を連続して戻る事故を防げる。
- 既存の MediaPipe Hands と静的 ES Modules の構成を維持できる。

## 採用しなかった案

- 人差し指ポーズへの遷移時だけ決定する案: 人差し指状態のままカーソルを移動したいという操作イメージに合わない。
- 人差し指ポーズを一定時間保持して決定する案: 即時反応を求めるユーザー決定に合わない。
- ピンチまたは従来のグーで決定する案: 今回選択された片手の指構成と一致しない。

## 影響と検証

- `src/gesture-rules.js`: 指ごとの伸展判定と2種類のポーズ判定。
- `src/gestures.js`: 戻るポーズの単発ラッチと旧合掌処理の撤去。
- `src/cursor.js`: カーソル基準点を人差し指先端へ変更し、決定ポーズ中は継続して対象を押す。
- UI・文書: 操作説明を新仕様へ更新。
- テスト: 指の組み合わせ、決定の連続発火、戻るの単発・再武装、旧入力の無効化を確認する。

実カメラでは、手の向きと距離を変えた認識、ボタンを横切る際の即時決定、戻るの1回発火を最終確認する。
